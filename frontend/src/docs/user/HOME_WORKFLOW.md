# Главная страница (Home) и поток работы

## Блок «Генерация» (вверху)

Сводные настройки одного запуска:

| Элемент | Назначение |
|---------|------------|
| **Развернуть / Свернуть** | Показать или скрыть поля: модель генерации, target model, шаблон домена, workspace, техники, кнопку доп. параметров. |
| **Режим фокуса** | Упрощённый вид: акцент на задаче и результате; боковые колонки (IDE, результат) можно скрыть. |
| **Модель генерации** | Модель, которая **пишет** промпт и reasoning внутри приложения. |
| **Target model** | Модель, **для которой** готовится итоговый промпт (влияет на подбор техник и формулировки). |
| **Шаблон домена** | Преднастройка под тип задачи (если доступны шаблоны). |
| **Workspace** | Подмешивание глоссария, правил стиля и сниппетов из выбранного профиля. |
| **Техники** | **Авто** — система подбирает техники; **Вручную** — вы выбираете набор в мультисписке (Ctrl+клик). |
| **Доп. параметры** | Температура, top-p, top-k, флаг **уточняющие вопросы** (см. ниже). |

## Степпер Brief → Spec / IDE → Prompt

Индикатор прогресса сессии:

| Этап | Смысл |
|------|--------|
| **Brief** | Есть формулировка задачи; превью и спецификация ещё в работе или пусты. |
| **Spec / IDE** | Заполнено превью структуры (цель, формат, ограничения, evidence и т.д.) до или после первого ответа модели. |
| **Prompt** | Сгенерирован готовый промпт (можно итерировать и сохранять). |

## Превью IDE (средняя колонка)

Структурированный разбор задачи: вкладки вроде **Spec**, **Intent**, **Issues**, **Evidence**. Это **не** обязательный ручной ввод — данные подтягиваются из анализа; помогает проверить, что система поняла задачу до финальной генерации.

## Уточняющие вопросы

Если в **Доп. параметрах** включено **«Уточняющие вопросы»** и контекста мало, модель может сначала вернуть блок вопросов, а после ваших ответов — итоговый `[PROMPT]`. Отвечайте в интерфейсе вопросов, затем продолжайте генерацию.

## Поле «Задача»

Кратко опишите цель, контекст, формат вывода и ограничения. Подсказки отображаются **плейсхолдером** в поле ввода (не занимают место отдельным блоком).

Кнопка **«Создать промпт»** активна, когда текст задачи не пустой (и нет блокирующей загрузки).

## Итерация

После появления промпта можно включить режим итерации: описать, **что изменить** в текущем промпте (сократить, добавить примеры, ужесточить ограничения и т.д.), и снова вызвать генерацию — система опирается на уже полученный результат.

## Сохранение и версии

Итоговый промпт можно **сохранить в библиотеку** (теги, название). История версий в рамках сессии отображается в блоке результата — удобно откатываться и сравнивать варианты.

## Передача задачи в Сравнение

Если в интерфейсе есть переход **на Сравнение с этой задачей**, текущий текст задачи подставится на страницу A/B — можно сразу сравнить два набора техник на одном вводе.
