# MetaPrompt — справка

**MetaPrompt** помогает проектировать промпты под конкретную задачу и целевую модель: структурирует запрос, подбирает техники из базы знаний, показывает разбор задачи (цель, формат, ограничения), при необходимости задаёт уточняющие вопросы и поддерживает итерации до сохранения в библиотеку.

Тексты совпадают со встроенной справкой в приложении (**Справка** в меню).

## Студия и простой режим

| | **Студия (Home)** | **Простой режим (/simple)** |
|---|-------------------|-----------------------------|
| Назначение | Спроектировать промпт под задачу | Быстро улучшить уже готовый текст промпта |
| Техники из YAML-базы | Да (авто или вручную) | Нет |
| Классификация задачи, Prompt IDE | Да | Нет |
| Как улучшают текст | Полный пайплайн генерации | Один вызов «редактора» с пресетами и мета-промптом из настроек |

Оба режима используют OpenRouter и ваши модели из настроек; лимиты те же.

## Типичный путь

1. Сформулировать задачу на главной (**Home**).
2. При необходимости ответить на уточняющие вопросы.
3. Посмотреть разбор задачи: цель, формат вывода, ограничения.
4. Нажать **«Создать промпт»**.
5. При желании итерировать или сохранить результат в библиотеку.

Альтернатива без техник и мастера: **Простой режим** — вставили текст промпта, нажали «Улучшить».

## Пробный режим и свой ключ

| Ситуация | Что доступно |
|----------|----------------|
| Без своего ключа OpenRouter | Лимит **пробных токенов** и ограничение по **стоимости моделей** (дешёвые completion). Подробнее — вкладка **User Info** и **Настройки**. |
| Своим ключом в **Настройках** | Список моделей из OpenRouter по вашему тарифу, лимиты хоста на запросы/сессию могут всё ещё действовать (защита сервера). |
