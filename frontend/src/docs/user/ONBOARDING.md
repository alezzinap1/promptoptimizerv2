# Онбординг

## Зачем это приложение

**MetaPrompt** помогает **формулировать и улучшать промпты** для языковых моделей: с учётом задачи, целевой модели, техник промпт-инжиниринга и (опционально) контекста workspace.

## Первые шаги

1. **Зарегистрируйтесь** или войдите. Сессия сохраняется в браузере; при истечении срока нужно войти снова.
2. Откройте **Настройки** и при необходимости укажите **свой API ключ OpenRouter**. Без ключа доступен пробный режим с лимитом токенов и ограничением по стоимости моделей.
3. В **Модели** отметьте **предпочитаемые** модели генерации и целевые — они появятся в выпадающих списках на **Home**, в **Сравнении** и в **Простом режиме**.
4. Загляните в **User Info**, чтобы видеть остаток пробных токенов (если ключа нет).

## Основные разделы приложения

| Раздел | Назначение |
|--------|------------|
| **Home** | Полный цикл: задача, превью IDE, уточняющие вопросы, генерация промпта, итерация, сохранение. |
| **Простой режим** | Вставили текст промпта — «Улучшить» — получили переписанный вариант (без выбора техник). |
| **Сравнение** | A/B: **одна** модель генерации и одна задача, **два** набора техник (A и B: авто или ручной выбор). Сравнение двух сгенерированных промптов. |
| **Библиотека** | Сохранённые промпты, поиск и теги. |
| **Техники** | Каталог техник и пользовательские переопределения. |
| **Метрики** | Локальная сводка событий и использования (для обзора активности). |
| **Workspaces** | Профили контекста: глоссарий, стиль, ограничения, сниппеты — подключаются на Home. |
| **Модели** | Справочник OpenRouter и предпочитаемые модели. |
| **Настройки** | Тема, шрифты, ключ API, параметры простого режима. |
| **User Info** | Использование токенов, пробный остаток, напоминание про свой ключ. |
| **Справка** | Эта документация. |

## Где настроить «простой» режим

**Настройки → Простой режим (улучшение промпта):** пресет по умолчанию и дополнительный **мета-промпт** (постоянные пожелания к улучшению).

Подробнее во вкладке **Простой режим** этой справки.
