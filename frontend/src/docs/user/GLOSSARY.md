# Глоссарий

**OpenRouter** — шлюз к разным LLM по единому API. Приложение вызывает модели через OpenRouter.

**Пробный режим** — работа **без вашего** ключа OpenRouter: действует лимит **пробных токенов** и ограничение по **стоимости** модели (completion за 1M токенов не выше порога, заданного на сервере). Подробности — **User Info** и **Настройки**.

**Генерационная модель** — модель, которая **пишет или улучшает** промпт **внутри** приложения (и выводит reasoning). Не путать с **целевой моделью**.

**Целевая модель (target)** — модель, **для которой** вы потом используете сгенерированный промпт. Влияет на подбор техник и формулировок в system/user контенте.

**Техника промпт-инжиниринга** — приём оформления инструкции (роль, few-shot, цепочка рассуждений и т.д.). На **Home** техники подбираются автоматически или вручную; на **Сравнении** задаются два независимых набора (A и B).

**Простой режим** — отдельный экран: один запрос к LLM, который **переписывает ваш промпт** по системным правилам, выбранному **пресету** и **мета-промпту** из настроек. Без многошагового мастера и без выбора техник из каталога.

**Мета-промпт (в настройках простого режима)** — дополнительные пожелания, которые добавляются к системному промпту улучшения (например: «всегда добавляй критерии успеха», «формат — маркированный список»).

**Workspace** — сохранённый профиль контекста: термины, правила стиля, ограничения, сниппеты. Подключается на **Home** в блоке генерации.

**Сессия (авторизация)** — привязка браузера к аккаунту после входа. Идентификатор хранится на клиенте и передаётся запросами к API; при **истечении срока** или **выходе** нужно войти снова.

**Prompt IDE / превью IDE** — структурированное представление задачи (цель, формат, ограничения, intent, issues, evidence) **до** или **вместе** с финальным промптом. Помогает проверить понимание задачи системой.

**Итерация** — повторная генерация с указанием, **что изменить** в уже полученном промпте (на Home после появления результата).

**Уточняющие вопросы** — опциональный шаг: модель сначала задаёт вопросы по недостающему контексту, затем выдаёт промпт. Включается в **Доп. параметрах** на Home.

**Reasoning** — текстовое объяснение модели генерации: почему выбраны техники и как устроен ответ (если включено в вашем сценарии).

**A/B сравнение (Сравнение)** — два промпта на одну задачу при **одной** генерационной модели и **разных** наборах техник (варианты A и B).

**Rate limit** — ограничение числа запросов к API за интервал времени на пользователя/IP (защита сервера). При ошибке «слишком много запросов» подождите или обратитесь к администратору инстанса.
