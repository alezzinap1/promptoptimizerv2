# Разделы приложения

Кратко: зачем заходить на каждый экран и что там делать.

## Сравнение (`/compare`)

- **Идея:** один и тот же **текст задачи** и **одна модель генерации**, два независимых набора техник — **A** и **B**.
- Для каждого варианта: режим **Авто** или **Вручную** (мультивыбор техник).
- Результат: два промпта, reasoning и краткое сравнение (победитель A / B / ничья) — чтобы выбрать удачный стиль техник под задачу.
- **Не путать** с «разными моделями на одном запросе»: здесь сравниваются именно **наборы техник** при той же модели генерации (если администратор не менял контракт API).

## Библиотека (`/library`)

Три вкладки в одном хабе:

- **Промпты** — сохранённые на **сервере** промпты из студии (поиск, теги, карточки).
- **Техники** — встроенный каталог и свои техники (как раньше отдельная страница).
- **Скиллы** — **только в браузере** (localStorage): инструкции для ассистентов. Сохранение из студии в режиме **Скилл** и кнопки «в библиотеку» для скилла попадают **сюда**, а не в серверные промпты.

## Техники (`/techniques`)

- Отдельная страница каталога; также пункт **Техники** в меню справа сверху.
- Просмотр **встроенных** техник из YAML-базы и **пользовательских** дополнений.
- На **Home** в ручном режиме техники выбираются из **выпадающего списка** с поиском (аналогично выбору workspace).

## Продуктовые метрики (**User Info**, внизу страницы)

- Старый маршрут `/metrics` перенаправляет на `/user-info#product-metrics`.
- **Локальная** аналитика: счётчики запросов, сохранений, воронка, задержки, типы событий.
- Обзор использования инстанса; биллинг OpenRouter — в кабинете провайдера; агрегаты по приложению — также в **User Info** сверху.

## Модели (`/models`)

- Справочник моделей **OpenRouter** (имена, идентификаторы).
- Отметка **предпочитаемых** моделей: они попадают в выпадающие списки на Home, Compare и в Простом режиме.
- Ограничения **пробного режима** (лимит токенов, порог цены модели) задаются на стороне сервера — при отказе генерации проверьте **User Info** и **Настройки** (свой ключ).

## Workspaces (`/workspaces`)

- Профили проекта: **название**, описание, предпочтительная target model.
- Поля контекста: **глоссарий**, **правила стиля**, **ограничения по умолчанию**, **опорные сниппеты** (по строке на элемент там, где так задумано в форме).
- На **Home** выбирается workspace в блоке генерации — его данные учитываются при сборке промпта.

## Настройки (`/settings`)

| Область | Что настраивается |
|---------|-------------------|
| Внешний вид | Тема, размер шрифта (если доступно в вашей версии). |
| API | Собственный ключ **OpenRouter** (опционально); без него — пробный режим. |
| Простой режим | Пресет по умолчанию, **мета-промпт** для улучшения промптов. |
| Прочее | Предпочитаемые модели могут дублироваться с страницы **Модели** в зависимости от версии UI. |

## User Info (`/user-info`)

- **Токены** и **оценочная стоимость** использования через приложение.
- Без своего ключа: **остаток пробных токенов** / лимит.
- Текстовый блок **«О сервисе»** (возможности) — из конфигурации бэкенда.
- Внизу страницы — **продуктовые метрики** (воронка, события, задержки).

## Справка (`/help`)
