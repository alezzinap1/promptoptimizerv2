# Prompt Engineer — Current Streamlit App

Этот файл описывает текущую рабочую demo-версию проекта.

## Назначение

`Streamlit`-приложение — это основной пользовательский интерфейс проекта на текущем этапе. Именно его стоит запускать для локальной проверки, демо и portfolio-показа.

## Запуск

```bash
pip install -r requirements.txt
streamlit run app/main.py
```

Открой `http://localhost:8501`.

## Переменные окружения

Создай `.env` в корне:

```env
OPENROUTER_API_KEY=sk-or-v1-...
DB_PATH=data/web_agent.db
```

`TELEGRAM_BOT_TOKEN` нужен только для legacy Telegram v1 и не обязателен для Streamlit-версии.

## Основные страницы

| Страница | Описание |
|----------|----------|
| `Home` | Генерация промптов, reasoning, итерации, история версий |
| `Сравнение` | A/B сравнение двух наборов техник |
| `Библиотека` | Сохранённые промпты, поиск, редактирование, экспорт |
| `Техники` | Справочник техник промптинга |
| `Метрики` | Локальная продуктовая телеметрия и usage signals |

## Что эта версия демонстрирует

- рабочий `prompt engineering workflow`;
- использование общей базы техник;
- генерацию под разные target models;
- версионирование и prompt library;
- направление дальнейшей миграции в `FastAPI + React`.

## Что не является целью этой версии

- production deployment;
- многопользовательская работа;
- зрелая web-архитектура;
- публичный SaaS runtime.
