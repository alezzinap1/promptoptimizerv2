# Продуктовое видение (кратко)

Сжатое резюме идей из архивного брейнсторма и их соответствие коду **на сейчас**.

## Тезис

Перейти от «сгенерировать промпт» к **проектированию и проверке**: intent, спецификация, прозрачное происхождение полей, диагностика слабых мест, при необходимости — стресс-тесты и адаптация под модели.

## Направления и статус

| Идея | В коде сегодня | Не сделано / сомнительно |
|------|----------------|-------------------------|
| **Evidence-Bound Prompting** | `core/evidence.py`, accept/reject в UI | Fine-grained provenance по каждой строке промпта |
| **Prompt Debugger** | `core/prompt_debugger.py`, список issues в IDE | LLM-as-critic как основной слой, auto-fix diff |
| **Intent Graph** | `core/intent_graph.py`, данные в превью | Интерактивный граф, отдельный graph UI |
| **Workspaces / Context Packs** | API + UI workspaces, глоссарий и правила | Загрузка документов, team sharing, mini-RAG |
| **Adaptation Matrix** | Подсказки под целевую модель (`target_model_cards`) | Один spec → несколько скомпилированных вариантов под разные модели |
| **Prompt Components / IR** | Нет отдельного IR | Только PromptSpec + строковая сборка в `ContextBuilder` — **идея не отброшена, но не начата** |
| **Scenario Lab** | Нет | Стресс-сценарии и прогоны — **в бэклоге, в коде нет** |
| **Reverse Forensics** | Нет | Разбор чужих промптов — **в бэклоге** |

## Что по смыслу «отброшено»

- **Telegram-бот и `bot/`** — не планируется в этом репозитории; упоминания удалены из кода.
- **Единственный UI на Streamlit** — заменён на React; Streamlit только архив в `app/`.
