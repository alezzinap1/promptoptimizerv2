# Demo Script

## Короткая подача

Продукт (MetaPrompt / Prompt Optimizer) — не просто вызов LLM, а помощник в **проектировании** промптов: классификация задачи, база техник, Prompt IDE (spec / evidence / debugger), итерации, библиотека, A/B техник.

## Demo flow на 5–10 минут

### Сценарий 1. Generate

1. Открыть **Home** (студия).
2. Ввести задачу, например:  
   `Нужен промпт для анализа финансового отчёта и возврата результата в JSON`.
3. Показать: классификацию, reasoning, итоговый промпт, метрики качества структуры.

### Сценарий 2. Iterate

1. **Итерировать** с обратной связью, например:  
   `Сделай промпт короче и добавь ограничения против галлюцинаций`.
2. Показать новую версию и историю версий в сессии.

### Сценарий 3. Save and reuse

1. Сохранить в библиотеку.
2. Открыть **Библиотеку** (`/library`).
3. Открыть сохранённый промпт обратно в студии.

### Сценарий 4. Compare

1. **Сравнение** (`/compare`) — одна задача, два набора техник (A/B).
2. Показать защиту от одинаковых наборов техник; при желании — judge.

### Сценарий 5. Метрики и usage

1. Открыть **User Info** (`/user-info`), прокрутить к **продуктовым метрикам** (прямой заход: `/metrics` → редирект с якорем).
2. Показать: генерации, сохранения, вопросы, задержки, недавние события (в зависимости от реализации UI).

## Для README / собеседования

- Structured output и доменные ограничения;
- Улучшение уже готового текста (**Простой режим** `/simple`);
- Сравнение техник для одной задачи;
- Локальная библиотека удачных промптов;
- Объяснение выбора техник (reasoning).
