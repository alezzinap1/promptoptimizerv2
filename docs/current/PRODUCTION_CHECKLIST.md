# Production Checklist

Go / no-go перед **публичным** продакшеном, а не перед локальной разработкой или закрытой бетой.

## Сейчас проект готов к

- portfolio demo;
- локальному использованию;
- controlled beta;
- демонстрации архитектуры.

## Сейчас проект не готов к (без доп. работ)

- публичному production launch «как SaaS для всех»;
- многопользовательскому SaaS без мониторинга и регламентов;
- платному доступу без биллинга и политик;
- открытому интернет-деплою без ограничений и аудита.

## Прогресс (высокий уровень)

- Фаза 1 (abuse / config / observability baseline) — в основном выполнена в MVP-объёме.
- Фаза 2 (auth / users / изоляция данных / путь к Postgres) — MVP выполнен.
- Фаза 3 (integration tests, runbook, дашборды) — **в работе / по ситуации**.

## Обязательный минимум перед production

### Auth and users

- модель пользователей;
- авторизация;
- разграничение данных по пользователям.

### Abuse protection

- rate limiting;
- лимиты размера ввода;
- бюджет генераций / токенов (в т.ч. trial при общем ключе);
- таймауты и защита от тяжёлых циклов.

### Storage

- стратегия данных (SQLite vs Postgres);
- бэкапы;
- миграции схемы при росте.

### Observability

- структурированные логи;
- мониторинг ошибок (например Sentry);
- health / readiness;
- при масштабе — дашборды latency и ошибок.

### Reliability

- тесты на core;
- по возможности интеграционные / e2e на ключевые flow;
- CI;
- повторяемый деплой.

### Config and deployment

- разделение dev/demo/prod;
- секреты вне репозитория;
- задокументированный процесс выката.

## Production decision

Переход в публичный production оправдан, если:

1. Продукт доказал полезность на demo / beta.
2. Выполнены критичные пункты чеклиста выше для вашей модели угроз (включая TLS, CORS, секреты, бэкапы).

**Основной UI** — **FastAPI + React**. Streamlit (`app/`) не является целевым контуром для новых фич.
