# План дальнейших улучшений

Приоритизированный бэклог. Актуальный продукт: **FastAPI + React**; Streamlit (`app/`) — архив.

## Высокий приоритет

1. **Полноценный Q&A flow** — завершить цикл уточняющих вопросов и повторной генерации в SPA (устойчивость к формату `[QUESTIONS]` со стороны модели).
2. **Метрики в продукте** — расширить то, что уже есть на **User Info** (когорты, воронки, явное «принял промпт», стоимость/сессия) при необходимости.

## Средний приоритет

3. **Version history parity** — при необходимости выровнять историю версий в web с тем, что было удобнее в Streamlit.
4. **Compare A/B** — явное baseline vs experimental, улучшить методологию «победителя»; опционально развить judge.
5. **Обработка 401** — при истечении сессии перенаправлять на логин без сырых ошибок.
6. **Semantic / embedding подбор техник** — опционально усилить авто-выбор техник (сейчас есть эвристика + LLM-классификатор; семантика частично используется в агентском роутере, не как общий ranker техник).

## Низкий приоритет

7. **Completeness score** — улучшить эвристику или явно задокументировать ограничения (не путать с product success).

## Подбор техник (идеи сверх текущего классификатора)

Сейчас: `core/task_classifier.py` (keywords) и опционально `core/task_llm_classifier.py`; `TechniqueRegistry.select_techniques` маппит типы и `complexity` на техники. Возможные усиления:

1. **Эмбеддинги описаний техник** — ранжирование по близости к задаче + фильтры по модели и бюджету контекста.
2. **Лёгкий классификатор по фичам** — без LLM: длина, доля кода, язык, маркеры формата, «размытость».
3. **Один дешёвый LLM-вызов только для маршрутизации** — JSON с `task_types`, `complexity`, `technique_hints` (частично пересекается с текущим LLM classifier).
4. **Использовать Prompt IDE** — `intent_graph`, `debug_issues`, заполненность `prompt_spec` для тонкой настройки набора техник.
5. **Bandit / feedback** — приоритет техник, чаще приводивших к сохранению в библиотеку (нужна аккуратная этика и объём данных).
6. **RAG по техникам** — при большом каталоге: поиск чанков → top-k в system prompt.
