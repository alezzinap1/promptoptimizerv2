# Анализ причин и спецификация интерфейса

## Часть 1. Логический разбор причин

### Проблема A: Темы и шрифты не работают — сбрасываются при переходе

**Шаг 1.** Где хранятся тема и шрифт?  
→ В `st.session_state["sb_theme"]` и `st.session_state["sb_font"]`.

**Шаг 2.** Кто их записывает?  
→ Виджеты `st.selectbox` на странице **Настройки** (`pages/5_Settings.py`) с `key="sb_theme"` и `key="sb_font"`.

**Шаг 3.** Кто их читает?  
→ `inject_styles()` в `shared_styles.py`:
```python
theme_id = st.session_state.get(theme_key, "slate")  # sb_theme
font_id = st.session_state.get(font_key, "jetbrains")  # sb_font
```

**Шаг 4.** Что происходит при переходе с Настроек на Home?  
→ Запускается только `Home.py`. Страница `5_Settings.py` не выполняется, её виджеты не рендерятся.

**Шаг 5.** Как Streamlit обрабатывает виджеты, которые не рендерятся?  
→ **Официальная документация (Widget behavior):**  
> *"If a widget command for a specific widget instance isn't called during a script run, then none of its parts are retained, including its value in st.session_state. If a widget has a key and you navigate away from that widget, its key and associated value in st.session_state are deleted."*

**Вывод:** При переходе на другую страницу Streamlit удаляет `sb_theme` и `sb_font` из `session_state`, потому что соответствующие виджеты больше не рендерятся. `inject_styles()` получает дефолтные `"slate"` и `"jetbrains"`.

---

### Проблема B: Фантомный сайдбар на миллисекунду

**Шаг 1.** Как сейчас скрыт сайдбар?  
→ Через CSS: `section[data-testid="stSidebar"] { display: none !important; }`.

**Шаг 2.** Где этот CSS?  
→ Внутри `<style>`, который вставляет `inject_styles()` через `st.markdown()`.

**Шаг 3.** Когда этот CSS попадает в страницу?  
→ После того, как Streamlit уже отрисовал layout. Сначала рендерится HTML с сайдбаром, затем выполняется скрипт страницы, затем в DOM добавляется наш `<style>`.

**Шаг 4.** Что видит пользователь?  
→ Сначала — стандартный layout Streamlit с видимым сайдбаром, затем — применение нашего CSS и скрытие сайдбара. Это и даёт короткую вспышку (FOUC).

**Вывод:** Сайдбар не убран из DOM, он только скрыт CSS. Пока наш CSS не загружен, сайдбар виден.

---

### Проблема C: Сайдбар всё ещё есть

**Факт:** Streamlit всегда рендерит сайдбар. Нет API, чтобы полностью его отключить.  
`initial_sidebar_state="collapsed"` только сворачивает его, но не убирает из DOM.  
Мы скрываем его через CSS, но элемент остаётся в разметке и может мелькать при переходах.

## Часть 2. Желаемый интерфейс (спецификация)

### 2.1 Общая структура

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  [Home] [Сравнение] [Библиотека] [Техники] [Настройки]     [Тема ▼] [Шрифт ▼] │  ← Верхняя панель (всегда видна)
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│                         КОНТЕНТ СТРАНИЦЫ                                    │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

- **Без сайдбара** — только main content.
- **Навигация** — горизонтальная полоса сверху, справа.
- **Тема и шрифт** — в той же верхней панели справа, доступны на всех страницах.

### 2.2 Страницы

| Страница | Содержимое |
|----------|------------|
| **Home** | Заголовок, expander «Генерация» (модель, целевая модель, домен, режим техник, доп. параметры), поле задачи, кнопка «Создать промпт», блок результата |
| **Сравнение** | A/B сравнение двух наборов техник |
| **Библиотека** | Поиск, фильтры, карточки сохранённых промптов |
| **Техники** | База знаний техник, фильтры |
| **Настройки** | Тема, шрифт (можно вынести в верхнюю панель и оставить здесь только подсказки) |

### 2.3 Требования к теме и шрифту

1. **Сохранение между страницами** — тема и шрифт не должны сбрасываться при навигации.
2. **Сохранение между сессиями** — при перезагрузке страницы настройки должны сохраняться (файл, cookie или другой механизм).
3. **Применение везде** — все страницы используют одну и ту же тему и шрифт.

### 2.4 Требования к сайдбару

1. **Отсутствие** — сайдбар не должен быть виден и не должен мелькать.
2. **Без FOUC** — не должно быть вспышки сайдбара при переходах.

## Часть 3. Решения (как добиться желаемого)

### 3.1 Темы и шрифты — персистентное хранилище

**Вариант A: Файл (рекомендуется)**

- Путь: `.streamlit/user_prefs.json` (или `data/user_prefs.json`).
- Формат: `{"theme": "forest", "font": "inter"}`.
- При смене темы/шрифта — запись в файл.
- При каждом `inject_styles()` — чтение из файла (с fallback на дефолты).
- Плюсы: работает между страницами и сессиями, не зависит от виджетов.

**Вариант B: Виджеты в entrypoint**

- Перенести селекты темы и шрифта в `main.py`.
- Рендерить их в `main.py` на каждом запуске (например, в верхней панели).
- Плюс: `session_state` сохраняется, т.к. виджеты всегда рендерятся.
- Минус: в `st.navigation` вывод `main.py` идёт в сайдбар, а сайдбар мы скрываем — виджеты окажутся невидимыми. Нужно либо показывать сайдбар, либо менять архитектуру.

**Вариант C: Placeholder key**

- Использовать `_sb_theme` для виджета и `sb_theme` для хранения.
- При `on_change` копировать `_sb_theme` → `sb_theme`.
- На каждой странице в начале: `st.session_state.setdefault("sb_theme", st.session_state.get("sb_theme", "slate"))`.
- Минус: при первом заходе на другую страницу (без Настроек) ключ всё равно может быть удалён. Нужно дополнительно сохранять в файл.

**Рекомендация:** Вариант A (файл).

---

### 3.2 Сайдбар — убрать вспышку

**Вариант A: Ранняя инъекция CSS**

- Добавить CSS скрытия сайдбара в `.streamlit/config.toml` через `[theme]` или кастомный CSS, если Streamlit это поддерживает.
- Проверить наличие опции `[client]` / `customCss` в актуальной версии Streamlit.

**Вариант B: Не скрывать, а использовать**

- Оставить сайдбар, но использовать его только для темы и шрифта.
- Убрать `display: none`, вернуть `initial_sidebar_state="expanded"`.
- Все остальные элементы — в main content.
- Минус: сайдбар занимает место, что противоречит желанию «без сайдбара».

**Вариант C: Смена фреймворка**

- Если Streamlit не даёт нужного контроля над layout, рассмотреть Gradio, NiceGUI, FastAPI + React/Vue и т.п.

**Рекомендация:** Сначала попробовать Вариант A (конфиг/ранний CSS). Если не получится — временно принять Вариант B (сайдбар только для темы/шрифта) или планировать миграцию (Вариант C).

---

### 3.3 Навигация

- Оставить `render_nav()` в main content каждой страницы.
- Убедиться, что `st.columns(5)` даёт достаточно места для подписей.
- При необходимости — вынести nav в общий layout (если Streamlit позволит).

## Часть 4. План пересборки

### Фаза 1: Персистентность темы и шрифта ✅ РЕАЛИЗОВАНО

1. Создан `app/user_prefs.py`:
   - `load_prefs()` → читает `data/user_prefs.json`
   - `save_prefs(theme, font)` → записывает в файл
2. В `inject_styles()` — тема и шрифт берутся из `load_prefs()`.
3. Тема и шрифт в верхней панели (nav) — при смене сохраняют в файл и вызывают `st.rerun()`.
4. Файл `data/user_prefs.json` в `.gitignore` (через `data/`).

### Фаза 2: Сайдбар и FOUC

1. Проверить, есть ли в Streamlit способ задать custom CSS до первого рендера.
2. Если есть — добавить CSS скрытия сайдбара туда.
3. Если нет — оставить текущий подход и документировать ограничение, либо перейти на использование сайдбара только для темы/шрифта.

### Фаза 3: Верхняя панель

1. Сделать единую верхнюю панель: nav + тема + шрифт.
2. Вынести её в общий компонент, вызываемый в начале каждой страницы.
3. Убедиться, что на всех страницах панель выглядит одинаково.

### Фаза 4: Тестирование

1. Смена темы на Настройках → переход на Home → тема сохраняется.
2. Перезагрузка страницы → тема и шрифт сохраняются.
3. Переходы между страницами → нет вспышки сайдбара (или она минимальна).

---

## Часть 5. Ограничения Streamlit

| Ограничение | Описание |
|-------------|----------|
| Сайдбар всегда в DOM | Нельзя полностью отключить, только скрыть CSS |
| Удаление ключей виджетов | При уходе со страницы Streamlit удаляет ключи неотрисованных виджетов |
| Порядок загрузки | Сначала layout, потом наш скрипт и CSS → возможен FOUC |
| Entrypoint и sidebar | В `st.navigation` вывод entrypoint идёт в сайдбар |

---

## Итог

1. **Темы/шрифты:** хранить в файле, читать при каждом `inject_styles()`.
2. **Сайдбар:** искать способ ранней инъекции CSS; если нельзя — либо принять вспышку, либо использовать сайдбар только для настроек.
3. **Интерфейс:** единая верхняя панель (nav + тема + шрифт), контент страниц в main area, без зависимости от сайдбара для основной функциональности.
