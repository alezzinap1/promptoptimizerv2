# Простой режим

Минимальный путь без **Home**, техник и IDE: только переписывание текста промпта.

## Идея

Минимум шагов: **вставили промпт → «Улучшить» → скопировали результат**. Без выбора техник и без многошагового мастера.

## Как это работает

1. Приложение собирает **системный промпт** из:
   - базовых правил редактора (не выдумывать факты, не менять язык, сохранять цель);
   - выбранного **пресета** (стиль улучшения);
   - вашего **мета-промпта** из настроек (если заполнен).

2. В **пользовательское сообщение** попадает только текст вашего промпта с инструкцией вернуть **один** финальный текст.

3. На экране **два столбца**: слева ваш **запрос** (редактируемый промпт), справа **ответ** модели; результат можно скопировать.

## Пресеты

| ID | Смысл |
|----|--------|
| `balanced` | Баланс ясности и краткости. |
| `shorter` | Сильное сжатие, убрать воду. |
| `stricter` | Жёстче ограничения и критерии. |
| `clearer_structure` | Секции, списки, явная структура. |
| `richer_examples` | По возможности — короткие примеры (без выдуманных данных). |

На экране простого режима можно **переопределить пресет** для одного запроса. Значение по умолчанию задаётся в **Настройках**.

## Мета-промпт в настройках

В **Настройки → Простой режим** укажите текст, который должен **всегда** учитываться при улучшении (тон, формат, доменные правила). Это не заменяет ваш исходный промпт — это слой поверх базовых инструкций.

## Модель

По умолчанию используется **первая модель** из вашего набора в настройках каталога моделей. На экране простого режима можно выбрать другую из того же списка.

## Ограничения

Действуют те же лимиты, что и для других вызовов API: размер текста, rate limit, пробный лимит токенов при отсутствии своего ключа.
