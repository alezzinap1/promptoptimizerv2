# Онбординг

## Зачем это приложение

Prompt Engineer помогает **формулировать и улучшать промпты** для языковых моделей: с учётом задачи, целевой модели и техник промпт-инжиниринга.

## Первые шаги

1. **Войдите** или зарегистрируйтесь.
2. Откройте **Настройки** и при необходимости укажите **свой API ключ OpenRouter**. Без ключа доступен пробный режим с лимитом токенов и дешёвыми моделями.
3. В **Модели** выберите предпочитаемые модели для генерации — они появятся в выпадающих списках на главной и в простом режиме.

## Основные разделы

| Раздел | Назначение |
|--------|------------|
| **Home** | Полный цикл: задача, уточнения, техники, итоговый промпт. |
| **Простой режим** | Вставили текст промпта — нажали кнопку — получили улучшенный вариант. |
| **Сравнение** | Два варианта промпта с разными наборами техник. |
| **Библиотека** | Сохранённые промпты. |
| **Техники** | Каталог и свои техники. |
| **Метрики** | Сводка по событиям и использованию. |
| **Workspaces** | Контекст проекта (глоссарий, стиль, ограничения). |

## Где настроить «простой» режим

**Настройки → Простой режим (улучшение промпта):** пресет по умолчанию и дополнительный **мета-промпт** (ваши постоянные пожелания к улучшению).

Подробнее: [Простой режим](SIMPLE_MODE.md).
