# План дальнейших улучшений

Приоритизированный список улучшений по результатам code review.

## Высокий приоритет

1. **Per-user API key** — хранить OpenRouter key в настройках пользователя, а не глобально в settings.json. Требуется изменение `services/api_key_resolver.py` и `services/settings.py`.

2. **Полноценный Q&A flow** — во frontend есть заглушка; завершить цикл уточняющих вопросов и повторной генерации.

3. **Metrics UI** — добавить product telemetry UI (события, сессии, использование).

## Средний приоритет

4. **Унификация core** — устранить дублирование между `core/` и `bot/core/` (task_classifier, context_builder, technique_registry). Либо удалить bot, либо вынести общую логику.

5. **Version history parity** — Streamlit-сценарий богаче; довести web-версию до паритета.

6. **Compare A/B** — добавить явное сравнение baseline vs experimental, улучшить методологию выбора победителя.

7. **Обработка 401** — при истечении сессии frontend должен перенаправлять на логин, а не показывать сырые ошибки.

## Низкий приоритет

8. **Обновить STREAMLIT_AUDIT** — документ устарел; отразить текущее состояние или пометить как архив.

9. **WEB_MIGRATION_PATH** — обновить статус: миграция завершена, web — основной продукт.

10. **Completeness score** — улучшить эвристику или документировать ограничения.
