# Demo Script

## Короткая подача проекта

`Prompt Engineer` — это не просто вызов LLM, а ассистент, который помогает проектировать более качественные промпты через классификацию задачи, базу техник и итерационный workflow.

## Demo flow на 5-10 минут

### Сценарий 1. Generate

1. Открыть `Home`.
2. Ввести задачу вроде:
   `Нужен промпт для анализа финансового отчёта и возврата результата в JSON`.
3. Показать:
   - классификацию задачи;
   - reasoning по выбору техник;
   - итоговый промпт;
   - prompt metrics.

### Сценарий 2. Iterate

1. Нажать `Итерировать`.
2. Добавить обратную связь:
   `Сделай промпт короче и добавь явные ограничения против галлюцинаций`.
3. Показать новую версию и историю версий.

### Сценарий 3. Save and reuse

1. Сохранить результат в библиотеку.
2. Перейти в `Библиотека`.
3. Открыть сохранённый промпт обратно в `Home`.

### Сценарий 4. Compare

1. Перейти в `Сравнение`.
2. Запустить A/B на одной задаче.
3. Показать, что проект не разрешает нечестное сравнение одинаковых наборов техник.

### Сценарий 5. Metrics

1. Перейти в `Метрики`.
2. Показать usage signals:
   - генерации;
   - сохранения;
   - вопросы;
   - latency;
   - последние события.

## Use cases для README / собеседования

- генерация промптов под structured output;
- улучшение существующего промпта;
- сравнение техник для одной задачи;
- создание локальной библиотеки удачных промптов;
- объяснение, почему под конкретную задачу выбраны именно эти техники.
