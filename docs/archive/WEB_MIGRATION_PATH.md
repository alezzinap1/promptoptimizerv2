# Web Migration Path (выполнено)

## Цель (историческая)

Переносить продукт в `FastAPI + React` после того, как `Streamlit`-версия зафиксировала product shape.

## Canonical flows

1. `Generate` — ввод задачи, классификация, техники, reasoning, prompt output, метрики.
2. `Iterate` — обратная связь и новая версия.
3. `Questions` — уточняющие вопросы и повторная генерация.
4. `Library` — сохранение, поиск, редактирование.
5. `Compare` — A/B техник при честно различающихся наборах.
6. `Metrics` — продуктовая телеметрия.

## Что расходилось со Streamlit (на момент документа)

1. Q&A flow — доработать в web.
2. Compare — избегать одинакового автоподбора на обе стороны.
3. Metrics UI — вынести в web (сейчас: блок на **User Info** / редирект `/metrics`).
4. Version history — при необходимости выровнять с богатым Streamlit-сценарием.

## Decision gate (исторический)

Не делать web основным, пока не готовы core-тесты и метрики — **пройдено** последующей разработкой.
