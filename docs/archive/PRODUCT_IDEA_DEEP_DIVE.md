# Product Idea Deep Dive — сжатая архивная копия

## Зачем был документ

Продуктовый фильтр: не размывать фокус случайными фичами, держать направление от «генератора текста» к **системе проектирования и проверки промптов**.

## Продуктовая гипотеза

Сместить фокус на: понять intent → закрыть пробелы → собрать prompt как систему → привязать к источникам и ограничениям → проверить устойчивость → адаптировать под модели.

## Идеи (суть одной строкой)

| Направление | Суть | Статус в коде (2026) |
|-------------|------|----------------------|
| **Intent Graph** | Граф неопределённостей вместо линейного Q&A | Частично: `core/intent_graph.py`, UI без полноценного графа |
| **Prompt Debugger** | Дефекты архитектуры промпта, severity, подсказки | Частично: `core/prompt_debugger.py`, issues в IDE |
| **Evidence-Bound Prompting** | Происхождение полей spec / accept-reject | **Внедрено:** `core/evidence.py`, UI Evidence |
| **Prompt Components / IR** | Сборка из слотов, компиляция в текст | **Не делали:** prompt остаётся строкой + PromptSpec, не отдельный IR |
| **Scenario Lab** | Стресс-сценарии, adversarial inputs | **Не делали** |
| **Adaptation Matrix** | Варианты промпта под разные модели | Частично: карточки целевой модели, нет матрицы «один spec → N вариантов» |
| **Context Packs / Workspaces** | Устойчивый контекст (глоссарий, стиль) | **Внедрено:** workspaces API + UI |
| **Reverse Prompt Forensics** | Разбор чужого промпта / итераций | **Не делали** |

## Волны (стратегия оригинала)

1. **Волна 1:** Debugger, Intent Graph, Evidence — формирование «IDE-ядра».
2. **Волна 2:** Components, Adaptation Matrix, Scenario Lab — инженерная глубина.
3. **Волна 3:** Context Packs (workspaces), Reverse Forensics — накопление и обучение.

## Что сознательно не брать в ближайший этап (оригинал)

- «Новости по промптингу», полноценная LLMOps-observability, full RAG, огромный workflow-builder без ядра — риск расползания.

## Итог оригинала

Ядро истории продукта: **Debugger + Intent Graph + Evidence**; остальное — надстройки. Часть уже отражена в текущем коде (см. колонку «Статус» выше).
