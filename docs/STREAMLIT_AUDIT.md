# Streamlit Feature Audit

Аудит фиксирует текущее состояние основной рабочей версии проекта.

## Работает

- `Home`: генерация промптов из задачи
- `Home`: итеративное улучшение существующего промпта
- `Home`: сохранение в библиотеку
- `Home`: история версий по сессии
- `Library`: поиск, редактирование, удаление, экспорт
- `Techniques`: просмотр базы техник, фильтрация и скачивание шаблонов
- `Metrics`: базовая локальная продуктовая телеметрия

## Частично работает

- `Home`: режим уточняющих вопросов
  - работает как цикл вопросов и повторной генерации;
  - все еще зависит от качества формата `[QUESTIONS]`, который возвращает модель.
- `Compare`: A/B сравнение
  - теперь блокирует нечестное сравнение при одинаковых наборах техник;
  - для сильного пользовательского сценария позже стоит добавить явное сравнение `baseline vs experimental`.

## Не стоит считать production-ready

- многопользовательский доступ
- auth и роли
- защита от abuse / rate limit
- внешний мониторинг
- централизованный deploy path

## Основные product risks

1. `Completeness score` полезен как эвристика, но не является доказательством реальной пользы промпта.
2. A/B сравнение требует более сильной методологии выбора варианта-победителя.
3. Q&A flow зависит от дисциплины форматирования ответа со стороны LLM.

## Решение на текущий этап

`Streamlit`-версию позиционировать как:

- честный `portfolio/demo product`;
- среду для проверки UX и product shape;
- источник требований для будущей web-миграции.
